This dedicated page explains the technical approach of the ElevateX project. The core system combines a SCAN/LOOK algorithm for directional lift movement, IR sensor based zone detection for a 3 to 5 second accessibility buffer, and a privacy-first design that avoids cameras and data storage.

Core technical pillars

The project technical approach is built around three main ideas: directional control logic, zone-based sensing, and privacy-preserving detection.

SCAN / LOOK algorithm

The lift tracks its current floor and movement direction. This allows the system to handle requests in a more organized sequence and avoid inefficient decisions while the car is already moving in one direction.

Zone detection

IR sensors form a waiting zone around pickup points. When a person is detected, the system creates a 3 to 5 second buffer so the passenger gets enough time to approach and board safely.

Privacy-first sensing

The system uses only presence or absence detection. It does not depend on cameras and does not store personal data, which keeps the model simple, low-risk, and privacy-aware.

System workflow

The technical flow turns input, sensing, and elevator movement into one understandable process from floor call to lift response.

1
Passenger request starts the process A floor call button triggers the system and sets the initial request that the controller must evaluate.
2
Controller checks direction and floor state The logic checks current floor position and present movement direction before accepting the next action.
3
IR zone sensing confirms passenger presence Sensor input detects whether someone is inside the waiting zone near the elevator pickup area.
4
Accessibility buffer is applied If a passenger is present, the system holds for 3 to 5 seconds to reduce the chance of skipping a slower-moving user.
5
Motor control executes movement The lift model moves using the motor and controller setup based on the processed logic decision.

Hardware understanding

The project model connects simple, low-cost components into a working smart-lift prototype for control, sensing, movement, and switching.

Controller + movement layer

Arduino Uno acts as the main controller, while the DC geared motor and motor speed controller handle the lift movement and speed regulation in the prototype setup.

Prototype component feel

The model uses practical hardware such as Arduino, IR sensors, buttons, wiring, power supply, relay support, and motor control to simulate elevator behavior.

IR SensorsDetect presence in the zone for accessibility-aware response.
DC MotorSimulates vertical elevator movement in the model.

Why this approach is strong

The technical approach is powerful because it balances efficiency, inclusivity, and privacy without making the system overly complex.

Good understanding

The whole model is easy to explain: input comes from buttons and sensors, the controller applies directional logic, and the motor executes the final movement decision.

Real-world relevance

The design directly addresses long queues, skipped passengers, and inefficient movement, which are practical problems in buildings and campuses.

Responsible innovation

By avoiding cameras and stored user data, the system remains respectful of privacy while still using intelligence to improve service quality.
